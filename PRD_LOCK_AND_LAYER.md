## 一、产品概述

### 1.1 游戏简介
麻将接龙消除游戏，玩家在棋盘上寻找相同牌面的麻将牌进行配对消除。棋盘支持最多 4 层牌叠落，牌的锁定/自由状态决定了玩家能否选中某张牌。

### 1.2 核心循环
1. 玩家查看棋盘，识别哪些牌是"自由"状态（可选中）
2. 点击两张相同牌面的自由牌进行消除
3. 消除后，被压住的牌可能解除锁定，变为自由牌
4. 重复以上步骤，直到所有牌消除完毕则过关

## 二、棋盘与牌的布局

### 2.1 交点网格系统（新设计）

棋盘的底屋坐标系是一个 **14×16 的交点网格**，而非 7×8 的格子。

- 原来的 7×8 格子，每个格子有 4 个角，这些角共享形成 14×16 个交点
- **所有牌的中心点只能放在交点上**
- 牌的坐标用 `(row, col)` 表示，其中 row ∈ [0, 13]，col ∈ [0, 15]


#### 交点与格子的对应关系

| 交点坐标 | 在原格子中的位置 | 对应原来的位置类型 |
|---------|----------------|----------------|
| row 偶数, col 做数 | 格子中心 | 正格（正压） |
| row 偶数, col 奇数 | 左右边中间 | 水平边位 |
| row 奇数, col 做数 | 上下边中间 | 垂直边位 |
| row 奇数, col 奇数 | 四个格子的角 | 角位 |


这样，正格/边位/角位不再是特殊情况，而是统一坐标系下的不同坐标。

### 2.2 占位规则（互斥区域）


每张牌是一个长方形，中心点放在某个交点后，它占据一个 **3×3 的互斥区域**。这 8 个不可放点对应牌的 4 个角和 4 条边的中点：

```
 角 边 角
 边 中 边
 角 边 角
```

- 同一层内，两张牌的 3×3 互斥区域不能重叠
- 不同层之间，同一交点不能被多层同时占据

### 2.3 多层系统

- 棋盘支持 **4 层牌**：Layer 0、Layer 1、Layer 2、Layer 3
- 每一层独立占据交点网格，层与层之间通过"覆盖"产生锁定关系
- 每一层的牌在视觉上有偏移，模拟立体叠落效果：
  - 横向偏移：`-10px × layerIndex`（上层相对底层向左偏移）
  - 纵向偏移：`-12px × layerIndex`（上层相对底层向上偏移）

### 2.4 底层牌（Layer 0）
- 底层牌只放置在**偶数行、偶数列**的交点上（row ∈ {0,2,4,...,12}, col ∈ {0,2,4,...,14}）
- 共 7×8 = 56 个位置，这些位置间距为 2，天然满足 3×3 互斥规则，不会互相冲突
- 共 56 张牌（28 对）

### 2.5 支撑规则

上层牌不能悬空放置，必须有下层牌支撑。

#### 支撑点的定义
一个交点能作为 Layer N 的支撑点，当且仅当该交点落在 Layer N-1 某张牌的 3×3 互斥区域内。

即：**支撑点 = 下层所有牌的 3×3 区域的并集**


#### 覆盖所有情况
不管底层牌怎么排列，支撑点的计算统一用并集：
- 边贴边 → 重叠区域是一条边上的 2~3 个点
- 角对角 → 重叠区域是 1~2 个点
- 四角汇聚 → 重叠区域是 1 个点
- 更复杂的多牌重叠 → 还是并集，每个点只算一次

#### 逐层递推
- Layer 1 的支撑点 = Layer 0 所有牌的 3×3 并集
- Layer 2 的支撑点 = Layer 1 所有牌的 3×3 并集
- Layer 3 的支撑点 = Layer 2 所有牌的 3×3 并集

#### 放置上层牌的前提条件
Layer N 的牌放在某个交点 `(r, c)` 上，必须同时满足：
1. `(r, c)` 是 Layer N-1 的支撑点（落在下层某张牌的 3×3 区域内）
2. `(r, c)` 与 Layer N 已有牌的 3×3 互斥区域不冲突

## 三、覆盖与锁定机制（核⼼）

### 3.1 什么是"锁定"
- **锁定的牌**：不可被玩家选中，视觉上显示为灰色蒙版（降低亮度）
- **自由的牌**：可被玩家选中，显示正常颜色

### 3.2 覆盖规则

上层牌会压住下层牌，产生锁定效果。覆盖的判定基于**交点坐标的空间重叠**：

#### 规则：一张牌被上层牌"覆盖"，当且仅当
- 上层牌的 3×3 互斥区域内，存在**相邻层**（N-1）的牌的中心交点
- 即：相邻两层的牌中心 `(r, c)` 与 `(r', c')` 满足 `|r - r'| ≤ 1 且 |c - c'| ≤ 1`
- **不跨层**：Layer 2 不会直接压住 Layer 0，只压住 Layer 1；Layer 3 只压住 Layer 2

**举例**：
- Layer 1 的牌放在 (6, 8)，那么 Layer 0 中所有中心在 `(5~7, 7~9)` 范围内的牌都被这张牌覆盖
- 这个规则自然统一了正压、边压、角压的效果——不需要单独判断位置类型

### 3.3 锁定条件

一张牌被锁定（不可选中）当满足以下**任一**条件：

#### 条件 A：被上层牌覆盖
- Layer N 的牌被 Layer N+1 的牌覆盖
- 被覆盖的牌直接锁定，不可选中
- 暗扣牌被锁定后，同样不可选中，不可翻开

#### 条件 B：同层左右同时被挡住
- 在同一层内，以当前牌的中心 `(r, c)` 为基准，检查其 3×3 互斥区**外扩一圈**的左右两侧各 3 个点：
  - **左侧 3 个点**：`(r-1, c-2)`、`(r, c-2)`、`(r+1, c-2)`
  - **右侧 3 个点**：`(r-1, c+2)`、`(r, c+2)`、`(r+1, c+2)`
- 左侧 3 个点中**任意一个**有同层的牌 → 左边被挡住
- 右侧 3 个点中**任意一个**有同层的牌 → 右边被挡住
- 左右**同时被挡住** → 该牌锁定
- **只挡住一边不算**：必须左右同时被堵
- **仅限同层**：不同层的牌不参与左右阻挡判定

#### 条件 C：已消除
- 已经被配对消除的牌（`matched = true`），不可再选中

### 3.4 解锁场景
- **上层牌被消除** → 覆盖关系自动解除，被压的牌可能变自由
- **相邻牌被消除** → 左右阻挡条件可能不再满足，牌变为自由

### 3.5 视觉表现
| 牌的状态 | 视觉效果 |
|---------|---------|
| 自由牌 | 正常颜色，可交互 |
| 被覆盖锁定 | 灰色蒙版（tint 0x999999） |
| 被左右夹住锁定 | 灰色蒙版（tint 0x999999） |
| 提示牌（自由） | 黄色高亮（tint 0xf1c40f）+ 摇晃动画 |
| 选中牌 | 绿色蒙版（tint 0x90ee90）+ 轻微放大 |

> 渲染深度、选中效果、菜单层级等视觉表现保持现有逻辑不变。

## 四、消除流程

### 4.1 选牌规则
1. 只能选中自由牌（未锁定）
2. 第一次点击 → 选中该牌（绿色高亮）
3. 第二次点击：
   - **同类型** → 进入消除动画
   - **不同类型** → 取消第一张的选中，选中新牌
4. 再次点击已选中的牌 → 取消选中

### 4.2 消除后状态更新
1. 两张牌标记为 `matched = true`
2. 重新计算所有牌的覆盖状态和自由状态
3. 检查是否还有可配对的自由牌：
   - 有 → 继续游戏
   - 没有 → 觸发智能洗牌

> 消除动画效果保持现有逻辑不变，多层牌之间的碰撞动画也沿用现有效果。

## 五、智能洗牌

### 5.1 触发条件
- 棋盘上没有仸何一对自由牌可以配对时，自动触发

### 5.2 洗牌规则
- 只改变未消除牌的牌面类型，不改变位置
- 保证每种类型成对出现
- 最多尝试 50 次，确保洗牌后**至少存在 3 对**可配对的自由牌
- 如果牌总数为奇数，移除一张最上层的牌（同时解锁其覆盖的位置）

## 七、关卡编辑器适配

### 7.1 坐标系切换
- 编辑器坐标系从 7×8 格子切换到 14×16 交点网格
- 放置点变为 14×16 个交点

### 7.2 支撑检测
- 编辑上层牌时，实时检测哪些交点有支撑（下层 3×3 幦集）
- 只有有支撑的交点才能放置上层牌

### 7.3 互斥检测
- 放置牌时自动检测 3×3 互斥区域冲突
- 冲突的交点不可放置

### 7.4 铺满当前层功能
- 提供"铺满当前层"按钮：
  - Layer 0：在所有偶数×偶数交点铺满 56 张牌
  - Layer 1/2/3：在当前层所有有支撑且不冲突的交点上铺满牌

### 7.5 其他
- 支持 4 层编辑（Layer 0 ~ Layer 3）
- 对称编辑（水平/垂直）基于新坐标系
