# 麻将接龙消除游戏 — 开发总结

> 乐信圣文 AI 方向岗位测试题
> 开发周期：2026年4月5日 — 4月7日
> 技术栈：HTML5 + Phaser.js 3.60.0 + PWA

**备注：**
- 实际用时约 **30 小时**（原计划 24 小时），主要花费在动态效果与音效的调试打磨上
- 大模型调用响应较慢，部分交互需要等待
- 整体成本约 **400 元**（主要为 API 调用费用）

## 一、项目概述

### 1.1 需求背景

收到乐信圣文 AI 方向岗位测试题，要求在 24 小时内完成一款类 Vita Mahjong 的麻将手游。

**核心需求：**
- 首页、游戏页、结果页三页面架构
- 支持至少 20 个关卡
- 手机设备可玩
- PWA 支持离线访问

### 1.2 方案选型

| 方案 | 优点 | 缺点 |
|------|------|------|
| Unity | 功能强大、可打包 APK | 构建耗时长、环境配置复杂 |
| **Web（Phaser.js）** | **开发快、即时预览、Vercel 一键部署** | 性能略逊于原生 |

**最终选择：Web 方案**

理由：24 小时时限内，开发效率优先。Phaser.js 是成熟的 2D 游戏框架，能满足麻将游戏的核心需求，且无需构建即可在手机浏览器测试。

### 1.3 技术架构

```
├── index.html          # 主入口 + PWA 配置
├── js/
│   ├── game.js         # Phaser 游戏初始化
│   ├── levels.js       # 关卡配置生成
│   └── scenes/
│       ├── home.js     # 首页场景
│       ├── game.js     # 游戏主场景
│       └── results.js  # 结果场景
├── levels/             # 20 个关卡 JSON 文件
└── assets/
    ├── pixelMajong/    # 麻将牌像素贴图
    ├── icons/          # SVG 矢量图标
    └── wav/            # 音效文件
```

## 二、实现思路

### 2.1 棋盘与牌的布局

采用 **14×16 交点网格** 系统，底层共有 **56 个位置**（28 对牌）。

- 底层位置坐标：偶数行 × 偶数列（row ∈ {0,2,4,...,12}, col ∈ {0,2,4,...,14}）
- 牌尺寸动态计算，适配不同屏幕
- 支持多层牌叠落（最多 4 层），上层相对下层有偏移
- 每张牌有独立的深度层级，通过公式计算：`depth = layer * 10000 + (row + col) * 100`

### 2.2 锁定与自由状态

牌能否被选中，取决于它的状态：

| 状态 | 说明 |
|------|------|
| 自由牌 | 可正常点击选中 |
| 被上层覆盖 | 被更高层级的牌压住，不可选 |
| 被左右夹住 | 同层左右两侧都被堵死，不可选 |
| 已消除 | 配对成功后，不可再选 |

### 2.3 配对与消除

1. 第一次点击自由牌 → 选中（绿色蒙版 + 放大）
2. 第二次点击：
   - 同类型 → 执行碰撞动画 → 消除
   - 不同类型 → 取消第一张选中，当前牌变为选中
3. 消除后刷新所有牌的锁定状态

### 2.4 智能洗牌

当没有任何一对自由牌可以配对时，自动触发洗牌：
- 保持牌的种类和数量不变
- 只改变位置
- 确保洗牌后至少存在 3 对可配对的自由牌

### 2.5 提示系统

- 默认 10 次提示机会
- 点击后高亮一对可匹配的牌（黄色蒙版 + 钟摆摇晃动画）
- 高亮持续存在，直到用户消除这对牌后才消失
- 提示的暗扣牌会自动永久翻正

## 三、核心功能开发

### 3.1 首页（HomeScene）

- 游戏标题
- 「开始游戏」按钮 — 进入第 1 关（或上次关卡）
- 「定制关卡」按钮 — 进入关卡编辑器
- 按钮音效反馈

### 3.2 游戏页（GameScene）

**顶部信息栏：**
- 返回按钮 — 二次确认后返回首页
- 当前关卡数
- 分数 — 每次配对 +200
- 剩余可匹配对数
- 菜单按钮 — 重新开始 / 返回主菜单

**棋盘区域：**
- 14×16 交点网格布局的麻将牌（底层 56 张）
- 支持多层叠落
- 入场动画（7 种随机效果）

**底部功能栏：**
- 提示按钮 — 显示剩余次数
- 重新洗牌按钮 — 显示剩余次数

**游戏逻辑：**
- 配对消除 + 碰撞动画
- 锁定/自由状态判定
- 智能洗牌
- 通关判定

### 3.3 结果页（ResultsScene）

- 过关成功：显示奖杯 + 夸奖语 + 击败百分比
- 过关失败：显示叉号提示
- 按钮：下一关 / 重新挑战 / 返回首页

### 3.4 关卡编辑器（EditorScene）

- 点位模式显示所有可放置位置
- 白牌工具 / 暗扣工具 / 橡皮擦
- 对称编辑（横向 / 纵向）
- 多层编辑（上层需下层支撑）
- 保存到 localStorage，游戏自动读取

## 四、遇到的问题与解决方案

### 4.1 布局坐标系设计

**问题**：最初设计棋盘布局时，考虑使用 7×6 的排布方式

**分析**：
- 7×6 = 42 个位置，21 对牌，数量偏少
- 无法充分利用屏幕空间，麻将牌会显得过大或过疏

**解决**：改用 **14×16 交点网格** 的设计

- 原 7×8 格子，每个格子有 4 个角，这些角共享形成 14×16 个交点
- 所有牌的中心点只能放在交点上
- 牌的坐标用 `(row, col)` 表示，其中 row ∈ [0, 13]，col ∈ [0, 15]
- 交点分为正格（偶数行偶数列）、边位（水平边、垂直边）、角位（四个格子的角），统一坐标系
- 底层牌只放置在偶数行、偶数列的交点（共 56 个位置，28 对）

**意义**：这个交点网格设计解决了边压边、角压角、正压等多种情况下的覆盖判定统一性问题，不需要为每种情况单独处理逻辑。

---

### 4.2 JavaScript 加载顺序错误

**问题**：页面黑屏，`HomeScene` 未定义

**原因**：场景文件在 `game.js` 之后加载，导致类定义不存在

**解决**：调整 `index.html` 中 script 标签顺序，场景文件先于 game.js 加载

### 4.2 变量作用域问题

**问题**：点击"开始游戏"无反应

**原因**：`createBoard()` 中使用 `h` 变量，但 `h` 只在 `create()` 中定义为局部变量

**解决**：将 `w` 和 `h` 改为 `this.w` 和 `this.h`，挂载到场景实例上

### 4.3 角压覆盖漏判

**问题**：角压牌没有被正确锁定

**原因**：角压的覆盖范围计算只包含对角线 2 个位置

**解决**：角压候选覆盖范围改为包含 2×2 网格全部 4 个位置

### 4.4 牌面类型不成对

**问题**：随机分配牌面可能出现奇数张同类型牌

**原因**：直接随机分配，没有保证成对

**解决**：改为"先生成成对类型再打乱"的分配方式

### 4.5 自定义关卡位置判断

**问题**：编辑器支持 .5 位置后，游戏内的碰撞检测不准确

**原因**：依赖 boardGrid 整数索引，无法处理 .5 位置

**解决**：改为基于真实像素距离判断左右是否被挡住，用包围盒重叠判断覆盖状态

## 五、视觉效果与动效

### 5.1 牌的选中效果

- 浅绿色半透明叠加层
- 以左上角为中心放大 10%
- 置顶显示（depth = 20000）

### 5.2 入场动画（7 种）

1. 左右交替飞入
2. 从上到下飞入
3. 从下到上飞入
4. 从右下角弹性放大
5. 从左下角弹性放大
6. 从右上角弹性放大
7. 从左上角弹性放大

### 5.3 碰撞消除动画

- 起飞阶段：两张牌向左右两侧飞起（偏移量自适应屏幕边缘）
- 靠近阶段：快速靠拢接触
- 缩小消失：向中心缩小并消失

### 5.4 洗牌动画

- 收拢阶段：所有牌弹性聚拢到中心
- 停顿展示
- 展开阶段：牌弹回原位

### 5.5 提示摇晃动画

- 黄色蒙版 + 钟摆式摇摆（-4° → +8° → -4°）
- 每 1200ms 执行一次

## 十、总结

这是一次在严格时限内（24 小时）完成的手游开发实践。通过选择 Web 方案，将时间集中在核心游戏逻辑而非环境配置，实现了：

- 完整的首页 / 游戏 / 结果三页面流程
- 20 个关卡的渐进式难度设计
- 多层牌叠落的锁定/自由状态机制
- 丰富的动画效果和音效反馈
- 关卡定制编辑器
- PWA 支持的手机端体验

麻将接龙消除的核心乐趣在于"寻找配对"和"解锁连锁"，通过多层牌结构和智能洗牌保证，每一局都能让玩家找到下一步的方向。
